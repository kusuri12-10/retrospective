## Migration이란? 

데이터, 애플리케이션, 서버, 시스템 환경 등을 하나의 플랫폼에서 더 나은 다른 환경으로 옮기는 과정

### 종류

1. 데이터 마이그레이션

하드웨어 디스크에서 최신 클라우드 스토리지로 데이터를 옮기거나, DB의 데이터를 다른 종류의 DB로 옮기는 작업

로컬 서버에 저장되던 사용자 프로필 이미지를 AWS S3로 이전하는 작업, oracle에서 postgres로 데이터를 옮기는 과정이 이에 해당한다.

2. 애플리케이션 마이그레이션

소프트웨어 프로그램을 한 환경에서 다른 환경으로 옮기는 작업

- 리프트 앤 시프트(재호스팅): 애플리케이션을 큰 변경 없이 한 플랫폼에서 다른 플랫폼으로 이동하는 과정
    - 예: 온프레미스(On-premise) 서버 $\rightarrow$ 클라우드(AWS, Azure) 환경

- 리팩토링(재설계): 새로운 환경에서 실행할 애플리케이션을 크게 변경하는 과정
    - 예: 기존 모놀리식 아키텍처를 마이크로서비스(MSA)로 전환

- 플랫폼 재구성

- 종료 또는 교체

3. 클라우드 마이그레이션

기업의 전체 IT 자산(데이터, 애플리케이션, 인프라)을 클라우드 환경으로 완전히 옮기는 대규모 작업

### 마이그레이션을 하는 이유?

- 기술 부재 청산: 오래된 환경에서 최신 환경으로 이동하여 서비스의 수명 연장

- 비용 절감: 값비싼 자체 서버 유지비 대신, 쓴 만큼만 내는 클라우드로 옮겨 운영 효율을 높임

- 성능 및 확장성: 사용자가 급증할 때 유연하게 대처할 수 있는 인프라로 이동하여 서비스 품질을 개선함

- 규제 및 보안: 최신 보안 표준 준수
