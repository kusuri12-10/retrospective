# inputstream vs bytearray

### 1. ByteArray (byte[])

- 특징
    - 데이터를 메모리에 한번에 올려 저장하는 자료구조
    - 전체 데이터를 메모리에 로드 
    
- 장점:
    - 읽기 속도가 빠름
    - 인덱스를 통해 빠른 접근이 가능

- 단점:
    - 대용량 데이터 처리 시 OutOfMemoryError가 발생할 수 있음

- 용도: 크기가 작은 데이터
    - 메모리 내 데이터
    - 이미지 버퍼

### 2. InputStream

- 특징
    - 파일, 네트워크 등 다양한 소스에서 데이터를 스트림 방식으로 순차적으로 읽는 추상 클래스
    - 데이터를 한 바이트 또는 버퍼만큼 조금씩 순차적으로 읽음

- 장점:
    - 데이터를 작은 덩어리(Chunk) 단위로 읽거나, 필요한 부분만 처리하여 메모리 효율이 좋음

- 단점: 
    - ByteArray에 비해 읽기 속도가 상대적으로 느림
    - 한 번 읽은 데이터를 다시 읽을 수 없음

- 용도: 크기가 큰 데이터
    - 대용량 파일 처리(S3 업로드/다운로드)
    - 네트워크 수신 데이터

### 3. 비교 요약

| 특징 | InputStream | byte[] (ByteArray) |
| --- | --- | --- |
| 로드 방식 | 스트리밍 (필요할 때 조금씩) | 메모리 로드 (전체) |
| 메모리 사용 | 낮음 (효율적) | 높음 (데이터 크기에 비례) |
| 데이터 크기 | 대용량 데이터에 적합 | 소형 데이터에 적합 |
| 접근 방식 | 순차적 | 인덱스 |
| 예시 | 파일, 네트워크, 파이프 | 메모리 내 데이터, 이미지 버퍼 |

### 결론

소형 데이터나 빠른 접근이 필요할 때에는 바이트 배열을, 이미지 파일을 올리는 등 대용량 파일을 처리할 때는 InputStream를 사용해야 한다.

이미지 업로드 처리를 할 때에는 InputStream를 사용하는 것이 효율적이다. 스프링에서는 [MultipartFile](20260321-MultipartFile.md)라는 InputStream 구현체를 이용해 파일 처리를 한다. 이 부분은 다음에 다루어보도록 하겠다.
